# PRD: Obsidian 本地同步系统

> 版本: v1.0 | 日期: 2026-06-29 | 状态: 已实现

---

## 1. 产品概述

### 1.1 问题陈述

用户在手机端（微信、浏览器等）发现有价值的内容（文章、视频、图片），通过"文件传输助手"转发给自己，但这些素材大量堆积，从未被整理、消化、利用。用户变成了"知识囤积者"。

现有的第三方工具（如"笔记同步助手"）虽然能解决同步问题，但**数据经过商业公司的云服务器**，存在隐私风险；且部分功能需要付费。

### 1.2 产品目标

构建一个**完全本地化、开源、免费**的"微信→Obsidian"知识同步管道：

1. **数据绝对安全**: 手机到电脑全程 P2P 加密传输，不经过任何第三方服务器
2. **快速捕获**: 在手机端一键分享，3秒内完成素材收集
3. **自动整理**: 电脑端自动抓取全文、提取视频字幕、AI生成标签/摘要
4. **开源透明**: MIT 许可，代码完全公开，任何人都可审计和自部署

### 1.3 目标用户

- **主要用户**: 用 Obsidian 做知识管理的人，注重数据隐私
- **次要用户**: 任何想要"手机快速捕获 → 电脑自动整理"工作流的人

## 4. 用户故事

### US1: 快速捕获公众号文章
> 我在微信里看到一篇好的公众号文章 → 点右上角「分享」 → 选择「Quick Capture」 → 手机上弹出"已保存" → 30秒后电脑 Obsidian 里出现这篇带全文的笔记。

### US2: 收藏B站视频
> 我在B站看到一个有价值的科普视频 → 分享到 Quick Capture → 电脑端自动抓取了视频标题+字幕 → Obsidian 里出现可搜索的文字笔记。

### US3: 保存图片素材
> 我在相册/微信里看到一张有用的截图 → 分享到 Quick Capture → 图片自动传到电脑 → Obsidian 笔记里嵌入了这张图片。

### US4: AI 自动整理
> (配置了 AI API Key) → 每次素材进入收件箱 → AI 自动根据内容打3-5个标签 → 写一段2-3句话的摘要 → 笔记归档到按日期组织的目录。

## 5. 技术约束

| 约束 | 说明 |
|------|------|
| 不使用任何云服务 | 同步用 Syncthing P2P，处理全部本地 |
| AI 为可选功能 | 不配 API Key 则跳过，不影响核心流程 |
| 仅桌面端 Obsidian | 插件 `isDesktopOnly: true` (需要 Node.js 文件系统能力) |
| Android 最低 SDK 26 | 覆盖 95%+ 设备 |
| Windows 防火墙 | 需要将网络设为"专用"模式，开放 Syncthing 端口 |
